# いわら保育園 Tシャツ注文集計システム v3

Tシャツトリニティの「デザインページ」から、各デザインに紐づく商品ページを取得し、
**デザイン → シャツの型 → 色 → サイズ** を職員が選べる注文ページです。

## 主な変更点 v3

- 管理画面に「全デザインの型・色・サイズを取得」ボタンを追加
- 各デザインページから商品ページ（例：ハイグレードTシャツ、ビッグシルエットTシャツ等）を取得
- 商品ページから色・サイズ・商品画像を取得
- 職員側フォームに「シャツの型」選択を追加
- 集計表に「型」を追加
- CSVにも `product_name` / `product_url` を追加

## 起動

```bash
npm install
npm start
```

- 注文ページ: `/`
- 管理画面: `/admin`

## 運用手順

1. Renderに再デプロイ
2. `/admin` を開く
3. 「全デザインの型・色・サイズを取得」を押す
4. 注文ページ `/` を確認
5. 職員にURLを共有

## 注意

Tシャツトリニティ側のHTML構造が変わると、取得処理の修正が必要になる可能性があります。
また、Render無料プランではSQLiteファイルが永続化されないため、本番運用では外部DBまたはPersistent Diskを推奨します。


## v6 追加機能

- 管理画面に「テスト注文データを全削除」ボタンを追加
- 注文一覧で1件ずつ削除できるボタンを追加
- 職員が注文送信したときに、送信完了のポップアップを表示


## v7 追加機能

- 管理画面の注文一覧に、各注文データを個別削除できる「削除」ボタンを追加
- 個別削除後、注文一覧・発注用集計・備考一覧が自動で再読み込みされる


## v8 修正

- 型ごとのサイズ取得処理を修正
- 商品ページ全体からサイズを拾う処理をやめ、サイズ選択欄の近く・option要素を優先して取得
- ビッグシルエットTシャツなど大人用シャツに、80/90/100などの子どもサイズが混ざる問題を防止
- 缶バッジ・アクリルキーホルダー等は FREE サイズとして扱う
